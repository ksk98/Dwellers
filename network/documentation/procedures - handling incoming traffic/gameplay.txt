PROCEDURE:      HANDLING GAMEPLAY
DESCRIPTION:    After successfully starting a game, we can now
                finally play the game (yay)!

A. Host tells clients that party is going to the next room.

HOST:           GAMEPLAY\r\n
                ACTION:NEXT_ROOM\r\n
                \r\n

B. Host tells clients that there is no next room, therefore dungeon is over.

HOST:           GAMEPLAY\r\n
                ACTION:DUNGEON_END\r\n
                \r\n

C. Host tells clients that combat is won.

HOST:           GAMEPLAY\r\n
                ACTION:WIN\r\n
                \r\n

D. Host tells clients that combat is lost.

HOST:           GAMEPLAY\r\n
                ACTION:DEFEAT\r\n
                \r\n

E. Send attack values

Client:         GAMEPLAY\r\n
                ACTION:ATTACK\r\n
                CONTENT-LENGTH:x\r\n
                \r\n
                object containing all hit data

F. Send rest action

Client/host:    GAMEPLAY\r\n
                ACTION:REST\r\n
                ID:x\r\n
                \r\n
